## Overview

| Parameter | Value |
|---|---|
| Full Name | 5G New Radio |
| Standard Body | 3GPP |
| First Deployment | 2018–2019 |
| 3GPP Release | Rel 15 (Phase 1), Rel 16 (Phase 2) |
| Peak DL Speed | **20 Gbps** |
| Peak UL Speed | **10 Gbps** |
| Real-World DL | 100 Mbps per user |
| Latency Target | **4 ms** (URLLC: <1 ms) |
| Connection Density | **1 million devices/km²** |

**Key philosophy shift:** Previous generations were pushed by what technology could do. 5G was **driven by specific use cases** first, then the technology designed around them.

---

## Three Use Cases — IMT-2020 (ITU Definition)

```
                    5G
                   / | \
               /     |     \
           eMBB   URLLC   mMTC
```

| Use Case | Full Name | Target | Examples |
|---|---|---|---|
| **eMBB** | Enhanced Mobile Broadband | High data rate | 4K/8K video, AR/VR, fast downloads |
| **URLLC** | Ultra-Reliable Low Latency Communications | <1 ms latency, 99.9999% reliability | Autonomous vehicles, remote surgery, industrial automation |
| **mMTC** | Massive Machine Type Communications | 1M devices/km², ultra-low power | IoT sensors, smart meters, agriculture |

Each use case has different requirements — 5G's **flexible numerology and network slicing** allow one physical network to serve all three.

## NSA vs SA — Deployment Modes

### NSA — Non-Standalone (most deployed today)
- 5G NR **radio** + **4G LTE core** (EPC)
- LTE anchor provides control plane; 5G NR is secondary for data (high throughput)
- Faster to deploy (reuses existing LTE infrastructure)
- **No true 5G core** → can't support all 5G features (network slicing, URLLC)

```
UE ─── gNB (5G NR) ─── EPC (4G core)
     └── eNB (4G LTE) ─┘
```

### SA — Standalone (target state)
- 5G NR radio + **5G Core (5GC)** — fully independent
- Supports all 5G features: network slicing, low latency, full service-based architecture
- Requires full 5GC deployment — more complex and expensive

```
UE ─── gNB (5G NR) ─── 5GC
```

**Why NSA first?** Operators could launch 5G service quickly by reusing LTE core investment. SA is the long-term target.

## 5G Core (5GC) Architecture

5GC uses a **Service-Based Architecture (SBA)** — network functions expose APIs to each other, like microservices.

### Key 5GC Network Functions

| NF | Full Name | 4G Equivalent | Role |
|---|---|---|---|
| **AMF** | Access and Mobility Management Function | MME | NAS signaling, mobility, authentication coordination |
| **SMF** | Session Management Function | P-GW (control plane) | Session setup, IP allocation, QoS management |
| **UPF** | User Plane Function | P-GW (user plane) + S-GW | Data forwarding, traffic measurement, QoS enforcement |
| **UDM** | Unified Data Management | HSS | Subscriber database |
| **AUSF** | Authentication Server Function | AuC | Authentication |
| **PCF** | Policy Control Function | PCRF | Policy rules |
| **NRF** | Network Repository Function | New | Service discovery between NFs |
| **NSSF** | Network Slice Selection Function | New | Selects network slice for UE |

**SBA interfaces:** HTTP/2 based REST APIs between NFs (vs point-to-point interfaces in 4G).

## 5G NR Radio (Air Interface)

### Frequency Bands

| Range | Name | Frequencies | Characteristics |
|---|---|---|---|
| **FR1** | Sub-6 GHz | 450 MHz – 6 GHz | Good coverage, moderate speeds; most initial deployments |
| **FR2** | mmWave | 24 GHz – 52 GHz | Very high speeds, very short range, line-of-sight sensitive |

**mmWave tradeoffs:**
- Pro: Huge bandwidth available → very high speeds
- Con: High path loss, doesn't penetrate walls, requires dense small cells

### Numerology (Flexible Subcarrier Spacing)

Unlike LTE's fixed 15 kHz subcarrier spacing, 5G NR supports multiple:

| μ (mu) | Subcarrier Spacing | Slot Duration | Use Case |
|---|---|---|---|
| 0 | 15 kHz | 1 ms | Similar to LTE, FR1 |
| 1 | 30 kHz | 0.5 ms | FR1 (most common 5G) |
| 2 | 60 kHz | 0.25 ms | FR1 and FR2 |
| 3 | 120 kHz | 0.125 ms | FR2 (mmWave) |
| 4 | 240 kHz | 0.0625 ms | FR2 reference only |

**Higher SCS = shorter slot = lower latency** — essential for URLLC at mmWave.

## Massive MIMO and Beamforming

### Massive MIMO
- 5G NR uses **32T32R, 64T64R** or more antenna elements
- 4G LTE used at most 8T8R (typical 4×4)
- More antennas → more spatial layers → higher capacity

### Beamforming
- Form narrow, focused beams directed at individual UEs
- Instead of covering whole cell equally, concentrate energy where UE is
- **Benefits:** Higher SINR per UE, lower interference to other UEs, better cell-edge performance
- **3D beamforming:** Both horizontal and vertical beam steering

### Beam Management in 5G NR
New concept not in LTE:
- **Beam sweeping:** gNB periodically sweeps beams across the cell
- **Beam measurement:** UE measures beam quality
- **Beam selection:** Best beam chosen for each UE
- **Beam failure recovery:** Fast recovery if beam is lost

## Network Slicing

**Concept:** One physical 5G network divided into multiple isolated virtual networks, each customized for a specific use case.

```
Physical 5G Network
├── Slice 1: eMBB (video streaming service)
├── Slice 2: URLLC (autonomous vehicle operator)
└── Slice 3: mMTC (IoT platform provider)
```

Each slice has:
- Its own QoS parameters
- Resource guarantees (not shared with other slices)
- Independent management

**Enabled by:** Virtualization (NFV — Network Function Virtualization) + Software Defined Networking (SDN).

## 5G NR Channels

### Downlink

| Channel | Purpose |
|---|---|
| **PDSCH** | Physical Downlink Shared Channel — user data |
| **PDCCH** | Physical Downlink Control Channel — scheduling; uses QPSK + polar coding |
| **PBCH** | Physical Broadcast Channel — MIB; part of SSB (Synchronization Signal Block) |

### Uplink

| Channel | Purpose |
|---|---|
| **PUSCH** | Physical Uplink Shared Channel — user data |
| **PUCCH** | Physical Uplink Control Channel — feedback (CQI, HARQ, SR) |
| **PRACH** | Physical Random Access Channel — initial access |

**Removed from 5G (was in LTE):** PHICH, PCFICH — HARQ feedback now more flexible.

### SSB — Synchronization Signal Block
- New in 5G NR — replaces LTE's CRS for initial cell access
- Contains: PSS + SSS + PBCH (MIB)
- Transmitted in **beams** → UE measures SSB RSRP per beam → selects best beam
- Periodicity: 20 ms (default)

## VoNR — Voice over New Radio

- Voice over 5G — equivalent of VoLTE for 4G
- Uses **IMS** as the session control framework (same as VoLTE)
- Requires: 5G NR + 5GC + IMS integrated
- In early deployments: fallback to VoLTE or 3G CS voice
- Long term: VoNR on SA 5G will be the standard

**Advantages of VoNR over VoLTE:**
- Lower latency (5G NR has <4 ms vs ~10 ms LTE)
- Can run simultaneously with high-speed data
- Foundation for AR/VR communication

## 5G vs 4G LTE Summary

| | 4G LTE | 5G NR |
|---|---|---|
| Peak DL speed | 100 Mbps (1 Gbps LTE-A) | **20 Gbps** |
| Latency | ~10 ms | **<4 ms** (URLLC: <1 ms) |
| Connection density | ~100K/km² | **1M/km²** |
| Base station | eNodeB | gNodeB (gNB) |
| Core | EPC | **5GC (SBA)** |
| Spectrum | Sub-6 GHz | Sub-6 GHz + **mmWave (FR2)** |
| Subcarrier spacing | Fixed 15 kHz | **Flexible** (15–240 kHz) |
| MIMO | Up to 8T8R | **Massive MIMO (64T64R+)** |
| Network slicing | No | **Yes** |
| Use case focus | Broadband | **eMBB + URLLC + mMTC** |
